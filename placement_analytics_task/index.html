Raw CSV vs Normalized DB

What we removed and why
Issue in Raw CSVOur FixImpact
18-digit IDs read as floats → silently rounded dtype=str on all ID columns No broken joins
Applications repeats name, email, phone per row Dropped — join via candidate_id Single source of truth
Interviews has no Application ID column Looked up via candidate_id + job_opening_id Correct logical link
Whitespace-only cells not NULL Global regex → NULL Clean aggregations
Enrolments FK to non-existent candidate Set FK = NULL (ON DELETE SET NULL) No FK violation crash
Applications with invalid candidate/job FK Row skipped entirely, warning printed Referential integrity preserved

How to JOIN These Tables

Correct join paths for common queries
Query GoalJoin Path
Student's full profile from enrolment enrolments → candidates via candidate_id
Which jobs a candidate applied to candidates → applications → job_openings
Interview details for a candidate candidates → applications → interviews
⚠️ Interviews → Applications join (Zoho) Must match BOTH candidate_id AND job_opening_id
Hired students with no recruiter workflow enrolments WHERE stage='Hired' + LEFT JOIN applications, check COUNT = 0
Top skills across all candidates candidates.skill_set split in Python or LIKE '%skill%' in SQL
Index tip: All FK columns are indexed (idx_enrolments_candidate, idx_applications_candidate, idx_applications_job, idx_interviews_application) for fast joins.
