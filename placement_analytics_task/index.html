Database Architecture Summary: To remove the redundancies in the raw data files, we normalized the tables to 3NF. Redundant contact info (email, phone, name) was eliminated from applications since it can be joined via candidate_id. In addition, the interviews table connects directly to applications using the application_id foreign key, representing the exact logical relationship (interviews occur in the context of an application).
